# URI

---

<b>URI(Uniform Resource Identifier)</b>

간단하게 리소스를 식별하는 것이라고 이해할 수 있습니다. <br>

URI 는 URL과 URN으로 나뉩니다.

- URL: Uniform Resource Locator
- URN: Uniform Resource Name

URL은 흔히 우리가 알고 있는 www.google.com 처럼 주소를 의미합니다. <br><br>
URN은 이름을 의미하는데, 예시로 영구식별자 등이 있습니다. (ISBN 등)<br>
그러나 URN으로는 해당 리소스를 찾을 수 없기 때문에, 일반적으로 URN은 흔히 쓰이는 개념은 아닙니다. 그래서 URI라고 하면 URL과 동일한 의미로 흔히 쓰입니다.

<br><br>

# URL 분석

--- 

https://www.google.com:443/search?q=hello&hl=ko
<br><br>

### https: 프로토콜
어떤 방식으로 리소스(자원)에 접근할 것인가에 대한 규칙입니다. 주로 http, https를 사용합니다.<br>
https 는 http에 보안이 추가된 것입니다.
<br><br>

### www.google.com: 호스트명
<br>

### 443: 포트 번호
http는 80포트, https는 443포트를 주로 사용합니다. 포트는 생략이 가능하기에, 주소창에서는 보통 생략됩니다.
<br><br>

### /search: 패스(경로)
<br>

### q=hello&hl=ko: 쿼리 파라미터
- key=value 형태입니다.
- ?로 시작하며, &로 추가합니다
- 해당 서버에서 리소스를 찾기위한 조건이라고 할 수 있겠습니다.

<br><br>

# 웹 브라우저 요청 흐름

---

1. 웹 브라우저에서 호스트명을 보고 DNS를 조회합니다 -> IP를 알아냅니다. <br>
2. 해당 IP주소에 HTTP 요청 메시지를 생성해서 TCP/IP패킷에 <u>패스</u>와 <u>쿼리정보</u>를 담아 보냅니다. <br>
3. 서버에서 HTTP 응답 패킷을 보냅니다. 이곳에는 HTML 문서가 담겨있습니다.
4. 웹 브라우저에서 HTML을 렌더링하여 결과물을 볼 수 있습니다.
