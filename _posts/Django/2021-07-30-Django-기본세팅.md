<h3>Django의 기본 세팅에 대해 알아보겠습니다.</h3>
<br><br>
우선 구름IDE에 들어가서 컨테이너를 만들어 줍니다. 

![구름IDE 생성](https://user-images.githubusercontent.com/55419868/127589937-14a46358-207c-4a92-9d7b-75b73d2b3462.JPG)

다른 설정들은 크게 상관은 없고, 소프트웨어 스택은 Python을 선택해 줍니다. <br>
저는 Github에 올리려고, 템플릿을 Github로 선택했습니다. <br>
Github에 올리실 생각이 없다면, 기본 템플릿을 선택하시면 됩니다. 

<br>
<br>

터미널을 보겠습니다.

```
root@goorm:/workspace/Django-mini-project# 
```
이런 문구가 떠있습니다. <br>
여기서 'Django-mini-project'는 컨테이너의 이름에 해당되니, 제각각일 것입니다.

이제 터미널에 명령어를 입력해 나가겠습니다.

```
mkdir mysite
```
> make directory의 줄인 말로 "mysite"라는 경로를 만듭니다. (폴더 생성)

```
cd mysite
```
> change directory의 줄인말로 "mysite" 경로로 이동합니다.

<br><hr>
<i>이제부터 'mysite'라는 폴더에서 Django를 시작할 것입니다.</i>
<hr><br>

```
pip3 install virtualenv
``` 

> virtual environment의 줄인말로, 가상환경을 설치합니다. <br>

<br><hr>
<h3>가상환경을 왜 설치하나요?</h3><br>
가상환경을 따로 만들지 않는다면, 그 컴퓨터에서의 환경에서 작업을 하게 됩니다. 그렇다면, 그 컴퓨터에 설치되어 있는 여러 패키지라던가 프로그램들의 버전이 이미 정해져 있습니다. <br>
이미 정해져 있는 환경에서 작업을 했기 때문에, 다른 환경에 간다면 공들여 작성한 코드가 실행이 되지 않을 수도 있습니다.<br>
그러나 가상환경을 만들어서 그곳에서 세팅을 하고 프로젝트를 만든다면, 다른 컴퓨터에서 작업을 하더라도 가상환경을 실행하여 작업한다면 정상적으로 코드가 작동할 것입니다. 
<hr><br>

```
pip3 list
```
> 설치되어있는 패키지를 확인하는 명령어로, 정상적으로 설치가 되었는지 확인해 줍시다.

```
virtualenv myvenv
```
> 가상환경을 생성하겠습니다. 'myvenv'는 이름으로, 아무렇게나 설정하셔도 상관없습니다. <br>저는 my virtual environment를 줄여서 이름지었습니다.

```
source myvenv/bin/activate
```
> 가상환경을 실행합니다.

<br><hr>
가상환경을 실행했으니, 이제부터 터미널이 이렇게 보일 것입니다.
```
(myvenv) root@goorm:/workspace/Django-mini-project/mysite# 
```
(myvenv)가 떠있어야 가상환경이 실행되었다는 뜻입니다. 
<br>이렇게 가상환경이 실행된 상태에서 진행하겠습니다. <br>
앞으로 컨테이너를 껐다가 다시 접속할 때, 두가지 명령어를 입력하고 시작해주세요. <br>
`cd mysite` <br>
`source myvenv/bin/activate` <br>
항상 가상환경에서 작업하셔야 합니다.
<hr><br>

```
pip3 install django
```
> 드디어 django를 설치합니다.

```
django-admin startproject landingpage .
```
> 띄어쓰고 .까지 꼭 찍어주셔야 합니다. 현재 폴더를 가르키는 뜻입니다.<br>
즉, 현재 폴더에 "landingpage"라는 프로젝트를 만드는 명령어입니다.

```
python manage.py migrate
```
> DB를 만지겠다는 뜻입니다. 입력해주세요.

```
python manage.py runserver 0:80
```
> 0:80 포트에 서버를 열겠다는 뜻입니다.<br>
> 서버를 종료하고 싶으면 `Ctrl` + `C` 를 누르시면 종료됩니다.<br> 서버는 평소에는 꺼두시고, 사이트에 접속하고 싶으실 때, 접속하시면 됩니다.


![실행 URL과 포트](https://user-images.githubusercontent.com/55419868/127592717-d4acdf1c-6a53-4bb5-b2d0-d7b32be41952.JPG)
프로젝트에서 실행 url과 포트로 들어가서 사이트에 접속하실 수 있습니다.


<br>
그러나 서버를 열어도 접속할 수가 없습니다. <br>
settings.py 의 ALLOWED_HOSTS 에서 허락을 해주어야 서버 접속이 가능합니다.


```
ALLOWED_HOSTS = ['*']
```
settings.py 로 들어가서 `ALLOWED_HOSTS` 변수를 찾아, 이렇게 입력해주세요. 모두에게 허락한다는 뜻입니다. (이건 터미널에 입력하는 것 아닙니다!)

<br><br>

![서버접속성공](https://user-images.githubusercontent.com/55419868/127592889-ab60ee8e-26f7-418f-bef7-1b49f338324b.JPG)

<br>
다시 서버로 접속해 보시면, 축하메시지와 함께 서버가 열린 것을 확인 할 수 있습니다!

```
python manage.py startapp main
```
> 메인이라는 앱을 만듭니다. 터미널에 입력해주세요.<br> 보통은 웹사이트 하나가 여러개의 앱들이 합쳐져서 만들어집니다.<br> 물론 하나의 앱으로도 만들 수 있습니다.

<br>
앱을 만들었다면, settings.py 에 앱을 등록해주어야 합니다. <br>

```
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'main',
]
```
settings.py 에서 `INSTALLED_APPS` 변수를 찾아 'main'을 추가해주겠습니다. (터미널에 입력하는 것 아닙니다!)

<br><br><hr>
"mysite"라는 폴더에서 시작하여, 가상환경을 만들고, 그곳에 django를 설치했습니다. 서버를 열고 앱을 만드는 것까지 해보았습니다.
